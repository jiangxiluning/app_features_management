# 后端部署文档

## 项目介绍

这是一个基于Flask的后端项目，提供API服务。本文档详细说明了如何部署该后端项目。

## 环境要求

- Python 3.7 或更高版本
- pip 3.0 或更高版本

## 部署步骤

### 1. 准备工作

1. 克隆项目代码到本地或服务器
2. 进入后端项目目录：
   ```bash
   cd backend
   ```

### 2. 配置部署参数

在部署前，您可以通过编辑 `config.yaml` 文件来配置部署参数：

```yaml
# 后端服务器配置
server:
  host: 0.0.0.0
  port: 5001

# 备份配置
backup:
  path: ./backups
  interval: 1
  keep_latest: 10
  enabled: true
```

### 3. 执行部署脚本

执行部署脚本以安装依赖：

```bash
./deploy.sh
```

脚本会自动：
- 检查并创建配置文件（如果不存在）
- 创建备份目录
- 安装项目依赖

### 4. 启动后端服务

部署完成后，启动后端服务：

```bash
python3 app.py
```

服务将在配置的端口上运行（默认为5001）。

### 5. 配置生产环境

在生产环境中，建议使用WSGI服务器（如Gunicorn）来运行Flask应用：

```bash
# 安装Gunicorn
pip3 install gunicorn

# 使用Gunicorn启动服务
gunicorn -w 4 -b 0.0.0.0:5001 app:app
```

## 配置说明

### 服务器配置

- `server.host`：服务器监听地址，`0.0.0.0` 表示监听所有网络接口
- `server.port`：服务器监听端口

### 备份配置

- `backup.path`：备份文件存储路径
- `backup.interval`：备份间隔，可选值：
  - 0: 1小时
  - 1: 1天
  - 2: 1周
  - 3: 1个月
- `backup.keep_latest`：保留的最新备份文件数量
- `backup.enabled`：是否启用自动备份

## 数据库说明

项目使用SQLite数据库，数据库文件存储在 `instance/app_knowledge.db`。

### 数据库初始化

首次运行时，系统会自动创建数据库表结构并添加默认管理员用户：
- 用户名：admin
- 密码：admin123

### 数据库备份

系统会根据配置的备份间隔自动备份数据库，备份文件存储在 `backups` 目录中。

## 常见问题

### 1. 依赖安装失败

- 检查Python和pip版本是否符合要求
- 检查网络连接是否正常，确保能正常下载依赖

### 2. 服务启动失败

- 检查端口是否被占用
- 检查配置文件是否正确
- 检查数据库文件权限是否正确

### 3. 数据库备份失败

- 检查备份目录是否存在且有写入权限
- 检查配置文件中的备份路径是否正确

## 开发模式运行

如果您需要在开发模式下运行项目：

```bash
python3 app.py
```

服务将在 `http://localhost:5001` 上运行。
