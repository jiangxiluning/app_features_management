# 0.9.1

**发布日期：** 2026-06-22  
**数据库 Schema：** 2.0.0（无变更，无需迁移）  
**类型：** PATCH — 导出能力增强与缺陷修复

---

## 概述

0.9.1 在 0.9.0 基础上完善功能导出体验：修复部分选择导出失效问题，新增功能树多选批量导出，并修复「导出功能属性」运行时错误。本版本**仅涉及前后端应用代码**，数据库 schema 不变。

---

## 变更范围

### 缺陷修复

#### 导出知识描述 — 部分选择无效

- **问题**：导出弹窗中取消勾选部分功能后，ZIP 仍包含应用下全部已审核功能。
- **原因**：前端已传递 `feature_ids`，后端 `POST /api/features/:app_id/export` 未使用该参数。
- **修复**：后端按 `feature_ids` 过滤待导出功能列表；ID 比较时做 `int()` 归一化。

#### 导出功能属性 — 点击报错

- **问题**：点击「导出功能属性」触发 `ReferenceError: Cannot access 'appNode' before initialization`。
- **原因**：`handleExportJson` 参数与 `try` 块内 `const appNode` 同名，内层声明遮蔽参数并进入暂时性死区。
- **修复**：内部变量重命名为 `treeAppNode`；过滤选中 ID 时统一 `Number()` 转换。

### 功能增强 — 功能树多选导出

在 [`Features.vue`](../../frontend/src/views/Features.vue) 功能树面板新增批量导出能力（管理员 / 开发者，`canExport`）：

| 交互 | 说明 |
|------|------|
| 功能节点 checkbox | 仅已审核（`approved`）功能显示；点击不切换右侧详情 |
| 导出工具栏 | 搜索框下方，显示已选数量、「清空」、两个导出按钮 |
| 功能节点右键 | 新增「导出知识描述 / 功能属性」；多选中导出全部选中项 |
| 应用节点右键 | **不变**，仍导出该应用下全部已审核功能 |

约束：

- 选中功能须属于**同一应用**；跨应用勾选提示「请选择同一应用下的功能」
- 搜索 / 「只显示支持引导」过滤变化时，自动清除不可见节点的选中态
- 知识描述：打开导出弹窗，预选树中已选 ID，仍可调模板与勾选
- 功能属性：直接下载 JSON，文件名含 `_选中N项` 后缀

## 版本更新方法

### 适用场景

| 场景 | 操作 |
|------|------|
| 从 0.9.0 升级 | **仅部署新代码**，无需数据库迁移 |
| 从 ≤1.1.0 且未执行 2.0.0 迁移 | 先按 [0.9.0 升级指南](0.9.0.md) 执行迁移，再部署 0.9.1 |
| 全新部署 | 按 [0.9.0 全新部署](0.9.0.md#一全新部署) + 本版本代码即可 |

### 从 0.9.0 升级步骤

#### 1. 停服（可选，建议低峰期滚动）

```bash
# 停止 Gunicorn / python app.py
```

#### 2. 备份（建议）

```bash
cd backend
python3 -c "from app import backup_database; backup_database()"
# 或复制 instance/app_knowledge.db
```

#### 3. 部署后端

```bash
cd backend
pip install -r requirements.txt   # 依赖无变更，可跳过
# 重启
gunicorn -w 1 -b 0.0.0.0:5002 app:app
```

#### 4. 部署前端

```bash
cd frontend
npm install    # 依赖无变更，可跳过
npm run build
# 将 dist/ 部署到 Nginx 或静态服务器
```

#### 5. 冒烟验证

- [ ] 功能树勾选 2 个功能 →「导出知识描述」→ ZIP 仅含 2 个 `.md`
- [ ] 功能树勾选 2 个功能 →「导出功能属性」→ JSON 数组长度为 2
- [ ] 应用节点右键导出 → 仍为全量
- [ ] 功能节点右键（未勾选）→ 仅导出该 1 项
- [ ] 跨应用勾选第二个功能 → 提示错误

```bash
cd backend
python3 run_plan_tests.py
```
