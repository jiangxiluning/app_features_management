# 0.9.0

**发布日期：** 2026-06-19  
**数据库 Schema：** 2.0.0（需执行迁移）  
**代号：** 方案 M+ — 多人数据一致性改造

---

## 概述

0.9.0 是首个面向多人协作生产可用的预发布版本，重点解决权限绕过、审核数据泄露、并发覆盖与前端数据陈旧等问题。在**继续使用 SQLite** 的前提下，引入 JWT 全覆盖、COW 审核、revision 乐观锁、SSE 实时通知及全量 schema 迁移。

---

## 变更范围

### 安全与鉴权（Wave 1）

- 全 API JWT 鉴权（除 `POST /api/auth/login`）
- 角色从 Token claims 读取，移除对请求体 `user_role` / `user_id` 的信任
- 备份/恢复/导入/用户管理限管理员
- JWT 有效期设为 8 小时
- 登录不再自动创建 admin，改由 `scripts/init_admin.py` 初始化

### 数据一致性（停服窗口核心）

- **COW 审核**：开发者写入仅产生 `audit_log`，不修改已发布 `feature` 正文
- **读侧 pending 隔离**：他人只见已发布快照；提交人/管理员可见草稿
- **revision 乐观锁**：管理员写入携带 `revision`，冲突返回 409
- **原子 approve**：`UPDATE ... WHERE status='pending'` 防重复审批
- **move 修复**：`before_content` 正确保存移动前 `parent_id`
- **删除修复**：管理员删除前清理 `audit_log`，兼容 `foreign_keys=ON`

### 数据库与迁移

- 新增 `feature.revision`、`feature.updated_at`
- `audit_log` 内容 JSON 标准化
- pending 数据回迁（对齐 COW 语义）
- 同级重名清理 + 唯一索引 `idx_feature_parent_name`
- 悬空 audit 清理
- SQLite WAL、`busy_timeout`、`foreign_keys` 生产化配置

### 实时协作（SSE）

- 新增 `GET /api/events` SSE 推送
- 后端写操作后广播 `data_change` 事件
- 前端弹窗提示「数据已更新」+「立即刷新」
- 过滤本人操作，避免自我通知

### 前端适配（Wave 3）

- `Features.vue`：revision / 409 处理；开发者保存后重载树
- `AuditApproval.vue`：手动刷新 + SSE 联动
- 移除 API 调用中的 `user_role` 参数
- `Home.vue` 统一建立 SSE 连接

### 部署变更

- Gunicorn 由 `-w 4` 改为 **`-w 1`**（SSE 内存广播要求）
- 备份改用 SQLite `backup()` API（含 `shutil.copy2` 回退）
- restore/import 补齐 `db.engine.dispose()` + 维护模式

### 延后项（不在 0.9.0）

- 前端轮询 / changes API（已由 SSE 替代通知方案）
- Blueprint 拆分
- 密码 bcrypt 升级

## 版本更新方法

### 适用场景判断

| 场景 | 操作 |
|------|------|
| 全新部署 | 跳过迁移，直接部署 + `init_admin.py` |
| 从 ≤1.1.0 升级 | **必须停服**执行 `migrate_to_2_0_0.py` |
| 已执行过 2.0.0 迁移 | 仅部署新代码 |

---

### 一、全新部署

#### 1. 后端

```bash
cd backend
pip install -r requirements.txt
# 可选：生产环境
pip install gunicorn

# 创建管理员（首次必须）
python3 scripts/init_admin.py admin <your_password>

# 开发启动
python3 app.py

# 生产启动（单 worker）
gunicorn -w 1 -b 0.0.0.0:5002 app:app
```

配置见 `backend/config.yaml`（默认端口 **5002**）。

#### 2. 前端

```bash
cd frontend
npm install
npm run dev          # 开发：http://localhost:3000
npm run build        # 生产：输出 dist/
```

开发环境 Vite 已将 `/api` 代理到 `http://localhost:5002`。

#### 3. 验证

```bash
cd backend
python3 run_plan_tests.py
python3 test_sse_e2e.py
```

---

### 二、从旧版本升级（停服维护窗口）

> **禁止**先部署新代码再跑迁移。

#### 步骤 1：公告停服

停止 Gunicorn / `python app.py` 及所有连接数据库的进程。

#### 步骤 2：备份与预检

```bash
cd backend

# 预览迁移报告（不写库）
python3 migrations/migrate_to_2_0_0.py --dry-run

# 审阅报告
ls migrations/reports/
```

#### 步骤 3：执行迁移

```bash
# 自动备份到 backups/pre_migrate_2_0_0_<timestamp>.db
python3 migrations/migrate_to_2_0_0.py
```

迁移内容：

1. DDL：`revision`、`updated_at`
2. 重复同级名自动重命名
3. audit JSON 化
4. pending 数据回迁
5. 唯一索引
6. 悬空 audit 清理
7. 写入 `schema_version = 2.0.0`

#### 步骤 4：迁移失败回滚

```bash
python3 migrations/migrate_to_2_0_0.py --rollback backups/pre_migrate_2_0_0_<timestamp>.db
```

#### 步骤 5：部署新代码

```bash
# 后端
pip install -r requirements.txt
python3 scripts/init_admin.py   # 仅当无管理员时

# 前端
cd ../frontend
npm install
npm run build
```

#### 步骤 6：启动与冒烟

```bash
cd backend
gunicorn -w 1 -b 0.0.0.0:5002 app:app
```

检查项：

- [ ] 管理员登录
- [ ] 功能树加载
- [ ] 开发者提交 → 管理员审核
- [ ] 双用户交叉操作，SSE 通知弹出
- [ ] 手动/自动备份

#### 步骤 7：开放服务

保留 `pre_migrate_2_0_0_*.db` 至少 30 天。

### 四、配置要点

**backend/config.yaml：**

```yaml
server:
  host: 0.0.0.0
  port: 5002
backup:
  path: ./backups
  interval: 1      # 0=1h, 1=1d, 2=1w, 3=1m
  keep_latest: 10
  enabled: true
```

**生产约束：**

- Gunicorn `-w 1`
- 单实例挂载 SQLite，禁止多机共享
- 监控 `database is locked` 日志

---

### 五、已知问题与注意事项

1. **测试隔离**：运行 `test_plan_consistency.py` 前会设置 `PLAN_TEST_DATABASE_URI`，不会操作生产库；请勿在旧版测试脚本上对生产库执行 `drop_all()`。
2. **SSE 与多 worker**：多 worker 下 SSE 通知仅到达同一进程连接的客户端。
3. **密码存储**：仍为 SHA-256 客户端哈希，生产环境请尽快修改默认密码。
4. **6 月 4 日前备份恢复后**：需重新执行 2.0.0 迁移（若 schema < 2.0.0）。
